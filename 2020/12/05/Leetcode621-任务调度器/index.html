Leetcode621-任务调度器
2020-12-05|算法
Word count: 1.9k|Reading time: 7 min

这是 Leetcode 621-任务调度器 题目的详细解答。

题目

给你一个用字符数组 tasks​ 表示的 CPU 需要执行的任务列表。其中每个字母表示一种不同种类的任务。任务可以以任意顺序执行，并且每个任务都可以在 1 个单位时间内执行完。在任何一个单位时间，CPU 可以完成一个任务，或者处于待命状态。

然而，两个 相同种类 的任务之间必须有长度为整数 \(n\) 的冷却时间，因此至少有连续 \(n\) 个单位时间内 CPU 在执行不同的任务，或者在待命状态。

你需要计算完成所有任务所需要的 最短时间 。

思路

方法一：模拟

思路

我们可以按照时间顺序，依次给每一个时间单位分配任务。

如果当前有多种任务不在冷却中，我们应当选择 剩余执行次数最多的那个任务。这样可以将每种任务的剩余执行次数尽可能平均，使得 CPU 处于待命状态的时间尽可能少。具体证明见：

https://leetcode-cn.com/problems/task-scheduler/solution/ren-wu-diao-du-qi-by-leetcode-solution-ur9w/

因此我们可以用二元组 \((nextValid_i, rest_i)\) 表示第 \(i\) 个任务，其中 \(nextValid_i\) 表示其因冷却限制，最早可以执行的时间；\(rest_i\) 表示其 剩余执行次数。初始时，所有的 \(nextValid_i\) 均为 1，而 \(rest_i\) 即为任务 \(i\) 在数组 \(tasks\) 中出现的次数。

我们用 \(time\) 记录当前的时间。根据我们的策略，我们需要选择的任务 \(i\) 具有如下两个性质：

  • 不在冷却中 ：\(nextValid_i\leq time\)
  • 剩余执行次数最多：\(rest_i\) 最大

遍历所有二元组就可以找到目标 \(i\)。之后，我们更新该二元组： \[ (nextValid_i, rest_i)\to (time+n+1, rest_i-1) \] 如果更新后的 \(rest_i=0\)，那么任务 \(i\) 全部做完，我们在遍历二元组时也就可以忽略它了。

而对于 \(time\) 的更新，我们可以选择不断增加 1 来模拟，但是这会在 CPU 待命时产生不必要的遍历。因此，我们在每一次遍历前，将 \(time\) 更新为所有 \(nextValid_i\) 中的最小值（如果它大于 \(time\)），从而「跳过」待命状态。

代码实现

复杂度分析

  • 时间复杂度：\(O(|tasks|\cdot |\sum|)\)，其中 \(|\sum|\) 是数组 \(task\) 中出现的任务种类。
  • 空间复杂度：\(O(|\sum|)\)。哈希表统计每种任务出现的次数，以及数组 \(nextValid\) 和 \(test\) 来统计各个任务的情况。

方法二：构造

思路

这是一个十分巧妙的方法。首先我们创建一个宽为 \(n+1\) 的矩阵，将执行次数最多的一种（或几种）任务按列放入矩阵中，如下图所示。

对于剩余的任务种类，按照如下规则放入矩阵：

  • 从倒数第二行开始放起；
  • 根据反向列优先的顺序放置任务，即：
    • 尽可能放在最大（最低）的行
    • 尽可能放在最小（最左）的列

这样我们的矩阵如下所示：

其实，矩阵代表了 CPU 运行任务的顺序，也就是矩阵中按行向后的顺序，矩阵中的空格代表 CPU 处于待命状态。这种构造方法有两种情况：

  1. 如果所有任务在 \(n+1\) 行内放完

    那么所有任务的最小总时间为： \[ (maxExec-1)(n+1)+maxCount \]

    其中 \(maxExec\) 为任务的最大执行次数，\(maxCount\) 为并列最大执行次数的任务种数。

  2. 如果放完所有任务后超过 \(n+1\) 行。

    因为 CPU 待命状态是为了等待任务的冷却时间，但是如果列数超过了 \(n+1\)，那么就算没有待命状态，人物之间的间隔肯定也大于 \(n\) 了。因此，待命状态都可以删除，最终的总执行时间就是任务的总数 \(|task|\)。

上面的过程看似复杂，但其实都只是思考的过程，不需要用代码实现。这些复杂的思考过程其实简单来说就是：

  • 如果我们没有填超过 \(n+1\) 列，那么我们的执行时间为： \[ \mathrm{ExeTime}=(maxExec-1)(n+1)+maxCount > |task| \]

  • 如果我们填了超过 \(n+1\) 列，那么我们的执行时间为： \[ \mathrm{ExeTime}=|task|>(maxExec-1)(n+1)+maxCount \]

由此，我们可以得到最核心的对于答案的解析式： \[ \mathrm{ExeTime}=\max\{(maxExec-1)(n+1)+maxCount,|task|\} \] 该式子适用于一切情况。

代码实现

复杂度分析

  • 时间复杂度：\(O(|task|+|\sum|)\)。
  • 空间复杂度：\(O(|\sum|)\)。

总结

这道题目要求较好理解，两种解答中，方法一可能是很多人能够想到的做法，更加常规，但其正确性却不易被严格证明；方法二的思路十分巧妙，通过抽象的数学推导，最终代码十分简洁。
